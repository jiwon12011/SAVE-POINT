## 0. 정체성 · 차별화 · 안 할 것

**나는 밤에만 문 여는 마녀 약사다.** 손님은 사람이 아니라 **의인화된 '마음의 결정'**(졸린 달·불안의 그림자·다 탄 재…) — 만질 수 있는 작은 정령. 약초를 빻고 섞어 알맞은 포션을 만들어 **잠재워 병에 봉인**하고, 잠든 정령들이 **선반(도감)에 식구처럼 쌓인다.**

> ⚠️ **순수 게임 픽션.** '결정'은 플레이어 기분 추적이 아니다. 매일 찾아오는 귀여운 손님/퍼즐. 자기돌봄·체크인 아님.

**차별화 한 끗 (Potion Craft와 가르는 핵심):**
- 표면 동사(빻기·섞기·데일리 손님)는 Potion Craft와 겹친다. **다른 점은 손님이 '주문서'가 아니라 조제에 개입하는 살아있는 존재**라는 것. 예: '불안의 그림자'는 가마 옆에서 액체를 흔들거나 탁하게 만들고, 플레이어는 빻기/섞기로 진정시켜 가며 조제한다 → **"환자가 곧 퍼즐 환경."**
- 이 "결정 개입"은 **2차 목표**지만, **데이터 모델에 지금 슬롯을 예약**한다(§4). 지금 결정을 단순 주문서로 못 박으면 나중에 못 끼운다.
- 정서 차별화: **봉인 = 재워 보내기.** 판매가 아니라 잠재움. 도감 = 잠든 정령들의 선반(카탈로그 아님).

**안 할 것 (지난 빌드가 '앱'이 된 이유):**
- ❌ 하단 탭 네비 + 덮는 모달 → ✅ **한 화면 = 조제대 무대**, 도감/멘토는 **밀어내는 슬라이드**(덮지 않음)
- ❌ 텍스트 토스트 피드백 → ✅ 표정·색변화·파티클·게이지
- ❌ 거대 데이터부터 → ✅ **수직 슬라이스(1×3×1)부터** 손맛 증명 후 데이터 확장
- ❌ 탭만 → ✅ 드래그·길게누르기
- ❌ 빨간 경고·독촉·실패 → ✅ 무압박(항상 완성, 품질만 변동)

## 1. 핵심 루프 (하드닝)

```
[결정 손님 도착] ── 상단 아치창에 떠서 발광 + 말풍선 요구
        │           요구 = 원하는 포션 + 오늘의 '변덕'(달게/차갑게/3번 안에…)  ★핵심
        ▼
[약초를 서랍→절구로 드래그] → [길게 눌러 빻기] (첫 입력! 즉각 과반응: 사각사각+진동+가루↑)
        ▼
[가루를 가마에 떨군다] → 가마 색이 실시간 블렌딩 → 정답 색에 가까울수록 가마가 은은히 빛남(근접 게이지)
        ▼
[색이 무르익으면 병으로 봉인] → 결정이 '결정별 고유 방식으로' 편안히 잠듦(치유 payoff)
        ▼
[포션이 선반에 꽂힘] 도감 +1 · 별사탕 · 마법사 멘토가 품질별로 다르게 반응
        ▼
[하루 손님을 다 재우면 → 불 끄기(마감)] 별사탕 정산 · 연속일 +1 · 내일 손님 예고 · 마녀의 일기 한 줄
```

**반복을 막는 핵심 = '변덕 요구(modifier)'.** 같은 '졸린 달'이라도 오늘은 *"조금 달게"*(달콤 약초 +1), *"차갑게"*(빻기 약하게/특정 약초), *"3번 안에"*(재료 수 제한)처럼 **수식어 1개**가 붙는다. 정답 레시피는 같아도 **매번 손이 다르게 움직여야 완벽**이 나온다. 콘텐츠 12종이 곱셈으로 체감 36~48종. → **"정답 외우면 끝"을 구조적으로 차단. 1차 필수.**

**첫 30초 후크 설계:** ① 첫 입력은 **빻기**(택타일 인과를 0.5초에 꽂음). ② **첫 손님은 약초 1개로 풀리는 쉬운 결정** → 30초 안에 치유 payoff 1회 완주. ③ 색 변화는 "예쁨"이 아니라 **정답 근접 게이지**(가까울수록 발광).

**재방문 동력:** 수집(선반 채우기) · 변주(변덕·달 위상) · 품질(완벽 노리기).

## 2. 한 화면 무대 (designer 확정)

**무대 배경 = `assets/backgrounds/bg-workshop-alchemy` 단일 확정.** 가마가 이미 중상단에 그려져 있어 UI는 그 위에 **히트존만** 얹는다("다 그리기"가 아니라 "그림 위에 손잡이 달기"). `home-bg-potion-counter`는 L자 구도라 무대 부적합 → 상점/홈으로.

세로 예산 (390×800, 가용 ~760px):

| 영역 | 높이 | 비고 |
|---|---|---|
| 상단 HUD | ~44px (6%) | Lv·별사탕. 얇은 글래스 바 |
| 결정 손님 | ~150px (20%) | 아치창에 떠서 발광 + 말풍선(가마 좌상단 회피) |
| **가마(주인공)** | ~230px (30%) | ⌀200 글래스 드롭존, 손님과 30px 의도적 겹침(깊이). 드롭존은 **하단 절반에 집중**(엄지 영역) |
| 절구 | **0 (가마 상판에 겹쳐 띄움)** | 독립 높이 회수 — 한 화면을 가능케 하는 핵심 트릭 |
| 약초 서랍 | ~150px (20%) | 하단 가로 스크롤 트레이(`bg-seed-drawer` 톤). 칩 56×72, 5개 노출 |
| 진입(도감·멘토) | ~60px (8%) | **하단 가로바 금지.** 좌우 모서리 **디제틱 오브젝트**(책 모서리=도감, 마법사 얼굴=멘토) |

→ 절구를 겹쳐 띄우고 **MVP에서 섞기·상점 진입을 빼면** 점유 ~72%로 **숨 쉬는 한 화면**이 된다.

**도감·상점은 모달이 아니라 '밀어내는 슬라이드'** (이게 안티패턴 재발을 막음):
- 진입점은 버튼이 아니라 **무대 위 오브젝트**(책 모서리/별사탕 항아리/마법사 얼굴)를 집어드는 디제틱 동작.
- **덮지 말고 밀어낸다:** 무대가 옆으로 `translateX`, 패널이 그 자리에 들어옴(88% 폭, 원래 무대 12% 노출). **백드롭 dim·우상단 X 금지.** 돌아가기 = 반대 스와이프/책 닫기. 전환 280ms ease-out, transform만.
- 도감 = `bg-potion-codex` 풀 슬라이드. 멘토 = dim 없는 **반쪽 말풍선 확장**(하단에서 올라옴).

## 3. 손맛 설계 (콘셉트의 심장)

| 동작 | 입력 | 보고 느끼는 피드백 |
|---|---|---|
| **빻기** | 절구에 올린 약초 **길게 누르기/문지르기** | 가루 게이지(scaleX) + 사각사각 파티클 + 진동, 완료 시 가루 무더기 |
| **넣기** | 가루를 가마로 **드래그** | 퐁당 파문 + 액체 색 실시간 블렌딩 + **정답 근접 시 발광** |
| **봉인** | 색이 무르익으면 **병으로 드래그/길게** | 빛 번짐 + 결정이 잠드는 연출 + 포션이 선반에 꽂히는 컷 |

- **치유 payoff = 결정별 고유 '잠드는 방식'**(졸린 달=하품하며 가라앉음 / 불안의 그림자=떨다 녹음 / 다 탄 재=잿빛이 꽃으로). payoff가 **수집의 일부**가 된다("이 결정 잠드는 거 보려고 또 만든다").
- **완벽 품질만** 화면 전체 별가루 등 **차등 연출**(품질 동력).
- **마법사 멘토 반응 다양성**: 완벽=embrace / 히든=thinking→놀람 / 평범=relax. 30개 표정 자산을 품질 피드백 채널로.
- **REDUCED 동등 경로 (처음부터):** 드래그/길게누르기 불가·모션 민감 → **탭 1회로 빻기/넣기/봉인.** 색을 직접 안 섞으므로 **targetHsl 자동 적용 = 색근접도 만점 처리** → 품질 동등(REDUCED가 저품질되는 차별 금지).

## 4. 데이터 모델 · 시스템 (developer 확정 — 빌드 전 못 박음)

**원칙: 콘텐츠(약초·결정·레시피)는 코드 상수, 세이브는 '진행'만** → 콘텐츠 추가 시 세이브 마이그레이션 불필요.

### 약초 HERBS — 색 기여는 **HSL**로(RGB 평균은 죽탕됨)
```
moonherb: { name:'달빛풀', hsl:[230,60,55], potency:1.0, sprite:'crops/moonherb.png' }
… 12종, 파일명 = id (매핑 0)
```

### 결정 CRYSTALS — 단순 주문서가 아니라 '개입' 슬롯 예약
```
sleep: { name:'졸린 달', silhouette:'crescent', sprite:'ailment-sleep.png',
         interfere:null /* 2차: 'shake'|'murk' 등 가마 개입 슬롯 — 지금은 비워둠 */ }
```

### 레시피 RECIPES — 결정 종속 + **집합 일치(순서 무시)** + 목표색
```
potion_calm: { name:'안식 포션', forCrystal:'sleep',
               ideal:['moonherb','still_lavender'], targetHsl:[235,55,55],
               sprite:'reward-potion.png' /* 도감 틴팅으로 차별화 — 아래 */ }
```

### 변덕 요구 MODIFIERS (★1차 필수)
```
{ id:'sweet', label:'조금 달게', effect:'+달콤 약초 1' }
{ id:'cold',  label:'차갑게',   effect:'빻기 약하게/특정 약초' }
{ id:'quick', label:'3번 안에',  effect:'재료 수 ≤3' }
```
손님 생성 시 결정 + 변덕 1개를 **dateSeed로 결정론적** 배정(같은 날 새로고침 = 동일).

### 색 블렌딩 규칙
- 가마색 = 넣은 약초들의 **HSL 가중평균**(potency 가중). **hue는 원형 최단경로 보간**(350°↔10°→0°). 매 pour마다 1회 계산 → CSS 변수 `--liquid`. (매 프레임 재계산 금지)

### 품질 점수식 (무압박 — 항상 완성)
```
quality = 0.6·색근접도 + 0.4·재료일치도   (+ 변덕 충족 가/감점)
  색근접도   = 1 − (가마색 ↔ targetHsl HSL거리 / 최대거리)
  재료일치도 = |넣은 ∩ ideal| / |넣은 ∪ ideal|   (자카드)
등급: ≥0.85 완벽 / ≥0.5 좋음 / 그 외 평범   ※ '정성(craftBonus)'은 2차 슬롯만 예약
```

### 도감 시각 차별화 (🔴 자산 리스크 해소)
`reward-potion.png` 1장뿐 → 레시피별 **포션 실루엣 1장 + 목표색(targetHsl) CSS 틴팅/마스크**으로 12종을 색으로 구분(+ HSL 색칩). → 콘텐츠 추가가 "데이터만"으로 유지됨.

### 세이브 (단일 키 `moonlit.save.v1`)
```
{ v:1, lv, exp, starCandy, streak,
  codex:{ potion_calm:{ discovered:true, best:'완벽' }, … },   // 영속
  today:{ dateSeed, customers:[{ crystal, recipe, modifier, status:'waiting|healed' }], closed:false },
  inventory:{ moonherb:5, … }, settings:{ reduced:false } }
```

## 5. 엣지케이스 (확정값)

| 케이스 | 처리 |
|---|---|
| 재료 부족 | 서랍에 0개면 약초 **흐리게+드래그 불가**. "무조건 완성"은 "있는 재료로는"의 뜻 |
| 틀린 조합 | **항상 포션 완성**(품질 평범). 결정은 "약하게 풀리며" 잠듦(찡그림 잔존). 실패 아님 |
| 중복 제작 | 허용, 도감은 **best만 갱신**. 별사탕 = 첫 제작 보너스 vs 반복 소액 |
| 빻다 만 상태 | MVP: 진행도 100% 전엔 **가마 투입 불가**(단순). 거친가루 약화는 2차 |
| 드래그 취소(드롭존 밖) | 약초 **서랍으로 스냅백**, state 롤백. 투입은 드롭존 안에서만 확정 |
| REDUCED | 색근접도 만점 처리로 **품질 동등** 보장 |

## 6. 재화 · 진행 · 재방문 · 내러티브

- **별사탕(단일 화폐):** 봉인·도감 해금·완벽 보너스 적립 → 약초/씨앗 구매(상점 2차).
- **마감(가벼운 정산):** 하루 손님을 다 재우면 **불 끄기 연출** + 별사탕 정산 + 연속일 +1 + 내일 예고. 세션의 시작·끝.
- **무압박:** 포션 못 만들어도 마감됨. 연속일 끊겨도 빨간 경고 없음(불이 작아질 뿐).
- **마녀의 일기 (가벼움, 1차 가능):** 손님을 보낼 때마다 한 줄씩 쌓이는 일기 = 수집 메타 + 내러티브 척추.
- **NPC 5인 = 손님 의뢰자(텍스트만, 1차):** 결정 뒤에 데려온 단골("내 단골이 불면증이라…") 한 줄 → 퍼즐이 미니 스토리. 자산 0 추가.
- **2차:** 시즌 보스('오래된 슬픔' — 작은 결정을 모아야 풀림, 엔드게임) · 조제실 꾸미기(공간 성장, 배경 27종 단계 해금) · 계절 달력 · 결정 진화/연쇄 · 결정의 가마 개입(차별화 한 끗) · 상점·재배·채집·히든 레시피.

## 8. 1차 플레이어블 (MVP — 범위 조임)

**목표:** 결정 3종 × 약초 5종 × 레시피 3개 + **변덕 요구 1종 이상.**
**착수 순서:** 첫 커밋은 **1×3×1 수직 슬라이스**(빻기→넣기→색→봉인→도감1칸 + REDUCED 동시)로 손맛부터 증명 → 돌면 데이터만 3×5×3으로 확장.

**포함:** 한 화면 조제대 · 빻기/넣기/봉인 3제스처 · 가마 실시간 색 · 정답 근접 발광 · 결정별 잠드는 payoff · **변덕 요구**(2일차 후크) · 도감 3칸(슬라이드) · 별사탕·마감 1회 · 마법사 멘토 품질별 반응 · 마녀 일기(선택) · REDUCED 동등.
**제외(2차):** 섞기(원 젓기) · 상점 진입 · 재배·채집 · 5인 의뢰 · 히든 · 결정 개입 연출(데이터 슬롯만 예약).

**완료 기준:** 빻고 섞어 결정을 재워 포션을 만드는 게 손으로 즐겁고, **2일차에 변덕 때문에 손이 다르게 움직이며**, 도감 한 칸 채울 때 작은 성취가 느껴지면 성공.

## 9. 기술 노트 · 성능 가드

- 바닐라 HTML/CSS/JS, 빌드 없음, 단일 localStorage. 콘텐츠=코드 상수, 세이브=진행만.
- 입력: **단일 PointerEvent 라우터 + 상태기계**(IDLE→DRAGGING→GRINDING→POURING→SEALING). `setPointerCapture`는 **고정 루트에** 건다(약초 엘리먼트 X). 무대 컨테이너에 `touch-action:none`(전체 body X — 패널 스크롤은 살림).
- 길게누르기 빻기: `setTimeout` 금지 → pointerdown→rAF 진행도 누적, move 거리 가산, up이면 진행도 보존.

### 성능 가드 (모바일 저사양 60fps — 필수 준수)
- 배경 PNG는 **WebP·논리폭 ≤828px로 변환, 한 장 ≤350KB.** 무대별 개별 로드, 현재+다음만 preload(전체 선로딩 금지). *(현재 864×1821·2MB×11=45MB → 사전 변환 필수, §11)*
- 가마 액체 베이스 색 = CSS 변수 RGB/HSL lerp만, `setProperty`는 매 프레임 아니라 **전이 시점·최대 30fps throttle**. 소용돌이/농도는 가마 내부 한정 소형 캔버스(≤256²), 풀스크린 캔버스 금지.
- 파티클은 **단일 rAF 루프·객체 풀 재사용.** 동시 캡 저사양 40/최대 60. 우선순위 분배(치유30/가마20/빻기10), 초과분은 신규 스폰 드롭.
- 이동·진행은 **transform/opacity만**(left/top/width 금지). 게이지 scaleX, 드래그 translate3d.
- `getBoundingClientRect`는 pointerdown 1회 캐시(리사이즈·회전·패널 열림 시 무효화). pointermove 내 레이아웃 읽기 0회 → 좌표만 저장, rAF에서 1회 적용.
- `backdrop-filter`는 **화면당 1개(가마 글래스)만.** 도감·상점 패널은 **가짜 글래스**(반투명 단색+1회 블러 배경). 움직이는 요소 뒤 backdrop-filter 금지.
- `will-change`는 드래그 중 요소에 pointerdown에 켜고 up에 해제(상시 금지).
- 첫 3초 평균 프레임타임 ≥18ms면 **자동 강등**(파티클 캡 절반·소용돌이 정지). `prefers-reduced-motion`이면 파티클 0·즉시 결과.
- 접근성: 색+아이콘+라벨 병기(품질=별1/2/3+라벨+테두리형태, 결정=고유 실루엣). 글래스 위 텍스트는 다크 플레이트(rgba(20,12,30,.55))+text-shadow, 본문 대비 4.5:1. 터치 타깃 44~48px+. 키보드/탭 동등.
