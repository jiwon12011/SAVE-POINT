# 1. 왜 SAVE POINT인가 (Why)

### 문제 정의: "기록해야 한다"는 강박이 사람을 지치게 한다

투두 앱을 열면 아직 못 한 일들이 쌓여 있다. 무드 트래커는 "오늘 기분이 어때요?"라고 묻지만, 정작 번아웃이 왔을 때 다섯 개 이모지 중 하나를 고를 기력이 없다. 루틴 앱은 "스트릭 3일째!"라고 축하하다가, 하루 빠지면 불길한 빨간 X를 찍는다.

기존 앱들이 공통적으로 범하는 실수는 하나다. **잘 해내는 날을 위해 설계됐다는 것.** 컨디션이 좋은 날엔 체크리스트가 도파민 회로를 자극한다. 하지만 사람들이 앱을 닫아버리는 날은, 컨디션이 좋지 않은 날이다. 피곤하고, 아무것도 하기 싫고, 오늘 내가 어디쯤 있는지조차 모르겠는 그날.

Habitica는 RPG 문법을 빌렸지만 할 일을 완료해야만 XP가 쌓인다 — 놓치면 HP가 깎인다. Finch는 귀여운 새가 돌봄의 대상이 되어 정서적 부담을 줄였지만, 결국 "오늘의 목표를 달성해야 새가 자란다"는 구조에서 벗어나지 못했다. 무드 트래커류(Daylio, Moodfit 등)는 감정을 수집하지만 **다음 행동을 제안하지 않는다.** 기록은 하는데, 나아지는 느낌이 없다.

결국 사람들이 앱을 그만 쓰는 이유는 복잡하지 않다.

> "잘 못 하면 쓰기 싫어지는 앱이라서."

SAVE POINT는 반대로 설계됐다. **망한 날에 오히려 켜고 싶은 앱.**

### 타깃 사용자: 두 개의 페르소나

**페르소나 1 — 지수, 25세, 디자인 전공 졸업유예생**
포트폴리오 마감이 3주 남았다. 매일 밤 투두를 작성하지만 아침엔 아무것도 하기 싫다. 루틴 앱 스트릭은 이미 끊겼다. Notion 대시보드는 공들여 만들었는데 열어보면 오히려 죄책감이 든다. 오후 2시에 유튜브 쇼츠를 두 시간 보다가 "나 왜 이러지"라는 자괴감과 함께 하루를 접는다. 서브컬처(게임, 애니, 웹툰)가 유일한 위안이지만 그것조차 "도피하고 있다"는 느낌을 지울 수 없다.
- **페인포인트:** 잘 못 하고 있다는 증거를 자꾸 기록하게 되는 앱들. "오늘도 실패" 서사의 반복.

**페르소나 2 — 민재, 29세, 중소 IT기업 3년차 개발자**
번아웃이 왔는지 안 왔는지 모르겠는 상태. 퇴근 후 소파에 누워 스마트폰만 보다 잔다. 자기관리를 하고 싶은 마음은 있지만 헬스·독서·영어공부를 동시에 시작했다가 2주 만에 전부 포기한 경험이 여러 번. 게임은 세계관에 몰입하는데 현실 루틴엔 그런 몰입이 없다. "오늘 뭔가 했다"는 감각을 잃은 지 오래됐다.
- **페인포인트:** 의지력을 전제하는 앱 구조. 컨디션이 나쁜 날엔 앱 자체가 부담. 게임은 재미있는데 현실 루틴은 왜 재미없는가.

### 사용 동기 · 트리거: 언제 이 앱을 여는가

| 트리거 | 상황 | SAVE POINT가 주는 1차 가치 |
|---|---|---|
| **아침, 오늘이 두려울 때** | 일어났는데 할 일이 무겁게 느껴짐 | 할 일 목록 전에 나의 HP·MP 먼저. "오늘 내 상태부터 보자"로 프레임 전환 |
| **번아웃 순간** | 오후에 아무것도 못 하다 자책 시작 | 상태창으로 번역되는 순간 자책이 분석으로. "나는 게으른 게 아니라 MP 고갈 상태" |
| **미루는 중** | 해야 할 걸 알면서 못 시작하고 딴짓 중 | 오늘의 최소 퀘스트 제안. "이것 하나만"의 허들을 낮춤 |
| **자기 전, 하루 정산** | 오늘 뭔가 했는지 모르겠고 허무한 밤 | 세이브 완료 화면. "오늘의 나를 저장"하는 의식으로 하루를 닫음 |
| **잘 안 풀리는 날** | 보스가 너무 크게 느껴질 때 | NPC 대사. 나를 아는 캐릭터가 맥락 있는 말을 건넴 |

### 경쟁 앱 대비 차별점

- **Finch 대비** — Finch는 새가 자라려면 결국 목표를 달성해야 한다(아무것도 못 한 날엔 제자리). **SAVE POINT는 아무것도 못 한 날의 체크인도 유효한 세이브다.** 저장이 곧 수행이고, 수행이 곧 회복이다.
- **Habitica 대비** — Habitica는 할 일을 안 하면 HP가 깎이는 패널티 구조 + 높은 진입장벽. **SAVE POINT는 패널티 없는 RPG.** 오늘 아무 퀘스트도 못 깼어도 상태창은 정직하게 남고 내일 다시 열 수 있다.
- **무드 트래커류(Daylio 등) 대비** — 감정 수집에 그치고 "그래서 뭘 해야 해?"에 답이 없다. **SAVE POINT는 상태 → 퀘스트 → 대사로 이어지는 행동 흐름이 있다.**
- **일반 투두·루틴 앱 대비** — 할 일 중심은 못 하는 날 죄책감의 원천이 된다. **SAVE POINT는 상태 중심 설계.** 상태를 먼저 진단하고 그에 맞는 최소 행동을 제안한다.

### 차별화 포인트 4가지

1. **"아무것도 못 해도 세이브는 된다" 철학** — 완료가 아니라 체크인 자체가 세이브. *왜 좋은가:* 이탈 없는 습관 형성. "오늘도 실패"가 아니라 "오늘도 저장했다"로 서사가 달라진다.
2. **금지 퀘스트** — "오늘은 이걸 하지 마라"는 부정 미션(야식 금지, SNS 1시간 이상 금지 등). *왜 좋은가:* 번아웃 회복엔 해야 할 것보다 하지 말 것이 더 실용적일 때가 많다. 기존 앱에 없는 발상.
3. **상태이상 태그 — 감정을 진단명으로** — "우울하다"가 아니라 "수면 부족", "과부하 Lv.2". *왜 좋은가:* 자책 언어를 기술 언어로 치환하면 감정적 거리두기가 된다.
4. **NPC 조력자 다양성** — 같은 상황에 힐러는 따뜻하게, 라이벌은 도발적으로, 여관주인은 담백하게. *왜 좋은가:* 사람마다 회복에 필요한 언어가 다르다. NPC 선택 자체가 자기인식의 한 형태.

### 핵심 카피 · 내러티브

**메인**
> "오늘의 나를 저장하고, 다시 시작할 힘을 얻는 곳."
> "잘 못 한 날도, 저장은 된다."

**스플래시 / 온보딩**
> *"모험가여, 잠깐. 지금 당신의 상태를 먼저 확인하겠습니다. 퀘스트는 그다음입니다."*
> *"여기는 판단 없는 세이브 포인트입니다. 오늘이 어떤 날이었든, 당신은 저장될 자격이 있습니다."*

**세이브 완료**
> **SAVED.** *"오늘의 [이름]이(가) 저장되었습니다. 내일의 모험은 여기서부터 시작됩니다."*
> *"HP 2 / MP 1이어도 괜찮습니다. 영웅도 여관에서 쉽니다."*

**아무것도 못 한 날 (퀘스트 클리어 0)**
> *"퀘스트 클리어: 없음. 그래도 오늘의 당신은 기록되었습니다. 이 세이브파일은 지워지지 않습니다."*

**NPC 대사 샘플**
> 힐러(번아웃): *"무리했구나. 오늘은 회복이 임무야. 가장 강한 모험가도 포션은 마셔."*
> 라이벌(회피): *"또 도망치려고? 뭐, 그것도 전략이긴 하지. 근데 내일도 거기 있을 거거든, 그 문제."*
> 여관주인(복귀): *"어서 오세요. 오늘 방은 비어 있어요. 뭐 드시겠어요? 아니면 그냥 앉아 계셔도 됩니다."*

# 2. 상세 사용자 흐름 + 로직·데이터 설계

## 2-1. 상세 사용자 흐름

### 첫 실행 흐름 (온보딩)

| 단계 | 화면/동작 | 진입 조건 | 이탈(완료) 조건 | 예외 처리 |
|---|---|---|---|---|
| 0. 스플래시 | 로고 + 세션 복원 | 앱 콜드 스타트 | 1.0초 경과 또는 세션 복원 완료 | 세션 만료 → 로그인, 오프라인 → 로컬 캐시 진입 |
| 1. 인증 | 로그인/회원가입 (Supabase Auth) | `session == null` | 세션 토큰 발급 성공 | 네트워크 실패 시 "게스트로 시작"(로컬 전용, 추후 병합) 허용 |
| 2. 온보딩 프로필 | 이름·직업 별명·자주 걸리는 상태이상·선호 NPC | `profiles.onboarded == false` | 필수 4필드 입력 + 저장 성공 | 별명 미입력 시 "모험가" 기본값, 선호 NPC 미선택 시 힐러 기본값 |
| 3. 첫 체크인 | HP/MP/집중력 + 상태이상 + 보스 | 프로필 저장 직후 자동 진입 | 스탯 3개 모두 1회 이상 선택 | 보스 텍스트 비움 허용 |
| 4. 첫 상태창 | 규칙 엔진 결과 + AI 대사 | 체크인 저장 성공 | "퀘스트 받기" 탭 | AI 실패 시 폴백 대사 즉시 렌더(3초 타임아웃) |
| 5. 첫 퀘스트 | 메인/서브/금지 퀘스트 카드 | 상태창에서 진입 | 카드 렌더 완료 | AI 실패 시 규칙 기반 기본 문구 |
| 6. 첫 세이브 | "오늘을 세이브" + 첫 세이브 연출 | 퀘스트 화면 도달 | 세이브 레코드 생성 + streak=1 | 퀘스트 0개 완료여도 세이브 가능 |

- **온보딩 1회성 보장:** `profiles.onboarded` 플래그. true면 2단계 영구 스킵.

### 매일 반복 루프 (앱 재진입 분기)

진입 시 위에서 아래로 먼저 매칭되는 분기를 실행한다.

| 판정 | 조건 | 라우팅 |
|---|---|---|
| A. 인증 | 세션 없음 | → 로그인 |
| B. 온보딩 | `onboarded == false` | → 온보딩 |
| C. 오늘 체크인 여부 | `daily_checkins`에 오늘 날짜 존재 | 있으면 → 오늘 상태창(재열람), 없으면 → 체크인 유도 홈 |
| D. 어제 세이브 여부 | `saves`에 어제 날짜 존재 | streak 유지 판정에 사용 |
| E. streak 연속성 | 마지막 세이브 = 어제 → 유지 / 그제 이전 → 끊김 | 홈 상단 배너 "연속 N일" 또는 "끊겼어요, 다시 시작" |

- **재방문 가드:** 같은 날 두 번째 체크인 시 "이미 체크인했어요. 덮어쓸까요?" 확인 → 덮어쓰기 시 체크인 업데이트 + 상태창 재계산.

### 분기·예외 처리

| 상황 | 처리 |
|---|---|
| 체크인 안 하고 홈 진입 | 홈을 "체크인 유도" 상태로 잠금(빈 상태창 + CTA). 퀘스트/세이브 비활성 |
| 보스 입력 비움 | 허용. 규칙 엔진 정상 동작(보스는 난이도 무관). AI는 "보스 없음" 모드로 **자기돌봄 메인 퀘스트**(휴식·정리·수분) 생성 |
| 퀘스트 0개 완료하고 닫음 | 세이브 정상. "오늘은 버틴 것만으로 충분" 메시지. streak 정상 증가 |
| 오프라인 | 체크인·규칙엔진·세이브 전부 로컬 동작. AI 대사만 폴백. `sync_queue` 적재 후 복귀 시 동기화 |
| AI 호출 실패/타임아웃(3초) | 규칙 기반 기본 문구 즉시 렌더. 백그라운드 재시도 1회. `ai_status='fallback'` 기록 |
| 게스트→계정 전환 | 로컬 레코드를 신규 user_id로 병합 업로드(날짜 충돌 시 서버 우선) |

## 2-2. 상태 → 난이도 규칙 엔진 (AI 없이 도는 뼈대)

### 4단계 점수 부여
각 스탯(HP/MP/집중력)을 동일 척도로 점수화한다.

| 단계 | 라벨(HP 예시) | 점수 |
|---|---|---|
| 1 | 거의 쓰러짐 | 0 |
| 2 | 버티는 중 | 1 |
| 3 | 그럭저럭 | 2 |
| 4 | 꽤 괜찮음 | 3 |

- **에너지 총점:** `base = HP + MP + 집중력` (0–9) → `energy = base − 상태이상차감` (하한 0 클램프)

### 상태이상 차감 규칙

| 규칙 | 차감 |
|---|---|
| 상태이상 1개당 | −0.5 |
| **중증 태그**(수면 부족·현실 도피·마감 공포) 1개당 추가 | −0.5 (중증은 합산 −1.0) |
| 상태이상 3개 이상 동시 | 추가 고정 −1.0 (과부하 페널티) |
| 차감 합계 상한 | 최대 −4.0 |

예: HP3·MP2·집중4(=6) + {수면부족, 알림과부하} → 차감 1.5 → energy = 4.5.

### 총점 → 난이도 모드

| energy 구간(반열린) | 모드 | 의미 |
|---|---|---|
| `[0.0, 2.1)` | **생존(Survival)** | 거의 방전. 회복이 유일 목표 |
| `[2.1, 4.6)` | **쉬움(Easy)** | 최소한만 |
| `[4.6, 7.1)` | **보통(Normal)** | 평범한 하루 |
| `[7.1, 9.0]` | **도전(Challenge)** | 컨디션 좋음, 보스 정면돌파 |

> 갭 없는 반열린구간(`>= 하한 && < 상한`)으로 정의해 경계값 미상 구간 제거. 입력 1–4 ↔ 점수 0–3 변환은 규칙으로 고정(§2-2 점수표).

### 모드가 결정하는 것 (전부 규칙으로 확정)

| 항목 | 생존 | 쉬움 | 보통 | 도전 |
|---|---|---|---|---|
| 메인 퀘스트 개수 | 1 | 1 | 2 | 3 |
| 서브 퀘스트 개수 | 0 | 1 | 2 | 2 |
| 금지 퀘스트 개수 | 2 | 1 | 1 | 0 |
| 퀘스트 시간 박스 | 5분 | 15분 | 25분 | 45분 |
| 보상 배율(별사탕) | ×1.5 | ×1.2 | ×1.0 | ×0.9 |
| 위험도(보스 직면) | 회피 | 우회 | 부분 직면 | 정면 |
| 추천 루트 | 회복 전용 | 정리·준비 | 본업 1블록 | 본업 다블록 |
| 추천 NPC | 힐러 | 여관주인 | 길드마스터 | 라이벌 |

- **보상 배율 역설계:** 컨디션이 나쁠수록(생존) 작은 행동의 가치를 ↑ 해 "조금 한 것"을 크게 인정. 도전은 ×0.9로 인플레 억제.

## 2-3. 보스 → 퀘스트 하이브리드 변환

### 규칙 vs AI 경계선

| 구분 | 규칙 엔진(결정론) | AI(Claude) |
|---|---|---|
| 난이도 모드 | ✅ | ✗ |
| 퀘스트 슬롯 구조(메인 N·서브·금지 개수) | ✅ | ✗ |
| 시간 박스·보상 배율·위험도 | ✅ | ✗ |
| 추천 NPC·추천 루트 | ✅ | ✗ |
| 메인/서브/금지 퀘스트 **문구** | ✗ | ✅ |
| NPC 회복 대사 | ✗ | ✅ |

**원칙:** 규칙이 "빈 슬롯이 있는 퀘스트 뼈대"를 만들고, LLM은 그 슬롯의 `title`·`flavor`만 채운다. LLM이 개수·시간·보상을 못 바꾸므로 일관성·예측가능성·오프라인 동작이 보장된다.

> 🔌 **제공자 독립:** 아래 구조는 어떤 LLM(Grok/xAI, Claude 등)으로도 동작한다. 모델명은 **자리표시자**이며, 실제 연결할 LLM이 확정되면 그 제공자의 구조화 출력·캐싱 방식에 맞춰 교체한다.

### LLM 호출 시점 / 모델 선택 (자리표시자)

| 시점 | 작업 | 모델(예시) | 근거 |
|---|---|---|---|
| 상태창 생성 시 | 퀘스트 슬롯 문구 일괄 생성 | 가볍고 빠른 모델 (예: `grok-*-mini` / `claude-haiku-4-5`) | 빠른 구조화 출력, 저비용. 하루 1회/유저 |
| NPC 대화 시 | 상태·퀘스트 반응 대사 | 풍부한 대화 모델 (예: `grok-*` / `claude-sonnet-4-6`) | 캐릭터성·문맥 반영 필요한 대화 |

- 퀘스트 생성은 **구조화 출력(JSON schema)**으로 강제 → 파싱 실패 0 목표. 스키마: `{ main:[{title,flavor}], sub:[...], forbidden:[...] }`.
- 대화는 짧은 응답(`max_tokens` ~512).
- ⚠️ **제공자별 차이:** 구조화 출력·프롬프트 캐싱의 API 형태는 제공자마다 다르다. LLM 확정 후 그 부분만 해당 제공자 SDK 방식으로 맞춘다(구조·프롬프트 설계는 그대로 재사용).

### 비용 절감

| 기법 | 적용 |
|---|---|
| **프롬프트 캐싱** | 시스템 프롬프트(세계관·NPC 말투·출력 스키마)를 캐시 고정 prefix. 모드·보스·상태만 가변부 |
| 캐시 안정화 | 시스템 프롬프트에 날짜·UUID 금지(캐시 무효화 회피) |
| 호출 최소화 | 퀘스트 3종을 **1회 호출**로 일괄 생성. NPC 대사는 사용자가 NPC 탭할 때만 lazy 호출 |
| 모델 분리 | 저빈도·풍부=sonnet, 고빈도·구조=haiku |

> ⚠️ **캐시 최소 토큰 주의:** 프롬프트 캐싱은 보통 캐시 프리픽스 최소 토큰 요건이 있다(제공자별 상이). 시스템 프롬프트가 그 이하면 캐시가 적용되지 않으므로, 세계관 설명을 충분히 채우거나 캐시 미적용으로 설계. (제공자 캐싱 미지원 시 이 항목 생략)

### 실패 폴백

| 단계 | 폴백 |
|---|---|
| 퀘스트 문구 | 모드별 사전 정의 템플릿(예: 보통 메인 = "오늘의 보스 [보스명]에 25분만 정면으로") |
| NPC 대사 | NPC별 고정 회복 대사 풀에서 모드 매칭 1개 선택 |
| 표시 | 사용자에게 실패를 노출하지 않음(자연스럽게 폴백) |

### NPC 대화 로직 / 추천 규칙
NPC는 현재 상태창(모드·energy·태그)과 생성된 퀘스트를 컨텍스트로 받아 반응한다.

| 조건 | 추천 NPC | 톤 |
|---|---|---|
| 생존 모드 OR HP=1 | 힐러 | 무조건적 위로·치유 |
| 쉬움 모드 | 여관주인 | 따뜻한 환대·쉬어가기 |
| 보통 모드 | 길드마스터 | 차분한 코칭·다음 한 걸음 |
| 도전 모드 | 라이벌 | 도발적 동기부여 |
| 결정 장애 태그 보유 | 마법사 | 선택지 정리·명료화 |

- 온보딩의 선호 NPC는 동률·근접 구간에서 **타이브레이커**로 우선.
- ⚠️ **MVP 폴백(NPC 2종):** 위 표는 5종 기준이다. MVP는 힐러·여관주인 2종뿐이므로, 길드마스터/라이벌/마법사를 가리키는 조건(보통·도전 모드, 결정 장애 태그)은 **MVP에서 여관주인으로 폴백**(또는 힐러). 존재하지 않는 NPC 추천 버그 방지. P2 확장 시 원래 매핑 복원.

| 데이터 | 로컬 캐시 | 서버 |
|---|---|---|

| 규칙 엔진 결과 | ✅(클라 계산, 즉시) | ✅ |
| AI 생성 퀘스트/대사 | ✅(표시 캐시) | ✅ |

- **규칙 엔진은 클라이언트·서버 동일 로직을 공유**해 오프라인에서도 단독 동작.

## 2-5. 보상 경제 연결

**별사탕(재화)**

| 적립 | 사용처 |
|---|---|
| 퀘스트 완료: `reward_stardust`(모드 배율) | NPC 대사 1회 더 듣기(리롤) |
| 세이브 1회: 고정 +10 | 도감 항목 즉시 해금(일부) |
| streak 마일스톤(7·30·100일): +50/+200/+500 | 상태창 테마·NPC 스킨 |

**도감 해금 조건(예시):** 칭호 "3일 연속 생존 모드 세이브 → '버티는 자'", 상태이상 도감 "해당 태그 누적 5회", 회복아이템 도감 "특정 NPC 대사 N회 / 특정 모드 첫 도달".

**연속 세이브(streak):** 어제 세이브 존재 → +1, 공백 1일+ → 1로 리셋. 같은 날 중복 세이브는 변동 없음(UNIQUE). `longest_streak = max(...)`.

**"세이브 = 완료 아님" 철학의 시스템 보장**

| 보장 지점 | 메커니즘 |
|---|---|
| 세이브 가능 조건 | 완료 수와 **무관**. `quests_completed == 0`도 정상 세이브 |
| streak 증가 조건 | 세이브 레코드 존재만 검사(완료 수 미참조) |
| 보상 분리 | 세이브 고정 +10은 완료와 독립. 완료 보상은 추가 레이어 |
| 메시지 | 0완료 세이브 시 "버틴 것만으로 오늘은 세이브 성공" |
| 데이터 검증 | `saves` INSERT에 완료 수 체크 제약 **없음**(의도적) |

# 3. 필요한 화면 전체

> 모바일 우선 · **귀여운 판타지(Cozy RPG)** · 파스텔 프로스트 글래스 · 치비 NPC · 따뜻한 밤 분위기
>
> ⚠️ 아래 화면별 "무드 노트"의 **구조**(무엇을 글래스로, 위계, 분기, 애니메이션 타이밍)는 그대로 유효하다. 단 **색·톤·아이콘 스타일은 §3 끝의 "디자인 시스템 요약(Cozy RPG)"을 기준**으로 읽을 것 — 네온/도트/다크퍼플 표현은 파스텔/둥근/따뜻한 밤으로 치환된다. (예: "라임 HP" → "분홍 하트 HP", "도트 테두리" → "둥근 모서리+말랑 그림자")

## 1. 스플래시
- **목적:** 앱 진입점. 세계관 몰입 + 로딩/세션 복원 대기.
- **구성:** 다크 퍼플 밤하늘 배경(도트 입자) · 중앙 세이브 크리스탈 로고(네온 글로우, 펄스) · "SAVE POINT" 타이틀 · 로딩 인디케이터 · 하단 감성 문구.
- **분기:** 첫 실행 → 온보딩 / 기존 → 홈 / 오프라인 감지 → 경고 배너.
- **무드:** 어두운 배경 vs 네온 글로우로 크리스탈이 '떠' 보이게. 여유 있는 수직 센터링, 안전 영역 고려.

## 2. 온보딩
- **목적:** 첫 사용자 기본 설정 수집(이름·직업 별명·자주 겪는 상태이상·선호 NPC). 3~4 스텝 폼.
- **구성:** 상단 프로그레스(1/4…) · 백버튼 · 단계별 질문 제목 · 입력/선택 UI(텍스트·태그·NPC 카드) · 하단 CTA("다음"/"시작하기").
- **분기:** 미작성 시 버튼 비활성 · 필드별 유효성 실시간 피드백.
- **무드:** 입력 필드/카드 글래스(알파 0.6) + 라이트 퍼플 스트로크. 필드 높이 48px+, 텍스트 16px+, 터치 타겟 44px+.

## 3. 홈 (메인 대시보드)
- **목적:** 매일 처음 열리는 화면. 오늘 상태를 한눈에 + 체크인 진입 + 어제 세이브 확인.
- **구성:** 헤더(날짜/요일 · 프로필) · 상태 패널(거대 HP/MP 바, 상태이상 배지, 위험도) · 오늘의 칭호(있으면) · 메인 CTA · 하단 단축 메뉴(퀘스트·NPC·기록) · 어제 세이브 미니 카드.
- **분기:** **체크인 전** = HP/MP '?' 회색, 메인 버튼 "오늘의 체크인 시작" / **체크인 후** = 실제 수치 애니메이션, 상태이상 태그, 추천 퀘스트 미리보기.
- **데이터:** HP/MP/집중력 %, 상태이상, 위험도, 오늘 칭호, 어제 세이브, 미완료 퀘스트 수.
- **무드:** 상태 영역 글래스(알파 0.65). HP=라임/MP=청자, 상태이상=주황/빨강. 진입 시 바가 0→최종값으로 차오르는 0.8초 연출.

## 4. 체크인
- **목적:** HP/MP/집중력 4단계 선택 + 상태이상 태그 + 오늘의 보스 입력 → 결과 화면으로.
- **구성:** 섹션별(HP·MP·집중력) 4단계 카드(아이콘+라벨, 단계별 색) · 상태이상 멀티셀렉트 태그 · 오늘의 보스 텍스트 입력 · 하단 CTA "상태 저장 & 다음".
- **분기:** 스탯 미선택 시 버튼 비활성 · 상태이상/보스는 선택사항(비워도 진행) · 선택 카드 하이라이트.
- **무드:** 선택 카드 글래스(미선택 0.5 → 선택 0.7), 선택 시 scale 1.02 + 글로우 200ms. 카드 높이 48px+.

## 5. 상태창 결과 ⭐ 포트폴리오 핵심
- **목적:** 체크인 데이터를 RPG 상태창으로 변환 표시. 칭호·위험도·추천 루트·회복 아이템 제시.
- **구성:** 글래스 상태창 패널(HP/MP/집중력 큰 바, 상태이상 배지, 위험도 게이지) · 칭호 영역 · 추천 루트(단계별 카드, 클릭 시 퀘스트로) · 회복 아이템 제안 · 하단 CTA "퀘스트 시작하기".
- **분기:** 위험도 70%+ → 붉은 틴트+경고 / 칭호 달성 → 골드 테두리 펄스 / 상태이상 3개+ → 경고 색.
- **데이터:** HP/MP/집중력 수치·%, 상태이상, 위험도, 칭호, 추천 퀘스트, 회복 아이템.
- **무드:** 상태창 패널 글래스 최강(알파 0.75~0.85) + 도트 테두리. 진입 시 떠오르듯 Fade+Scale(600ms), 게이지 순차 채움(300ms 간격).

## 6. 퀘스트
- **목적:** 메인 보스·서브 퀘스트·금지 사항을 탭/섹션으로 관리. 완료 체크.
- **구성:** 헤더(진행도 "2/5") · 탭(메인/서브/금지/완료) · 메인 퀘스트 큰 카드(체크박스+보스 마크) · 서브 퀘스트 스택 카드 · 금지 사항 섹션(경고 톤) · 하단 CTA "세이브 완료"(부분 완료 허용).
- **분기:** 완료 카드 회색+취소선 · 금지 항목 토글(위반 기록) · 완료 포함/제외 토글.
- **무드:** 퀘스트 카드 글래스(0.6, 메인 0.7), 금지 사항 붉은 틴트(0.5). 완료 시 체크 바운스(200ms)+페이드.

## 7. NPC 상담소
- **목적:** 현재 상태 맞춤 NPC 추천 목록 → 선택 시 대화 시작.
- **구성:** 헤더("누구의 말을 들어볼까요?") · 추천 사유 문구 · NPC 카드 스택(도트 아바타+이름+한 줄 소개+대화 통계) · 필터(추천만/모두).
- **분기:** 상태 분석 기반 추천 순서 · 첫 진입 시 온보딩 선호 NPC 최상단 · 대화 기록 유무로 통계 표시.
- **무드:** NPC 카드 글래스(0.65 → 호버 0.75 + scale 1.02). 카드 높이 120px+. 진입 시 아래→위 슬라이드.

## 8. 대화 ⭐ 포트폴리오 핵심
- **목적:** 선택 NPC와 회복 대사를 주고받는 RPG 텍스트 어드벤처. 선택지 2~3개.
- **구성:** 헤더(NPC 이름 · 닫기) · NPC 도트 일러스트(160×160) · 글래스 말풍선(타이핑 효과 권장) · 선택지 버튼 세로 스택 · (선택) 대화 히스토리.
- **분기:** 첫 대사(상태 감지 멘트) → 선택지별 분기 → 마지막 대사(회복 격려 + "저장할까요?") → 세이브 완료로.
- **데이터:** NPC 이름·일러스트, 현재 대사(AI/폴백), 선택지 배열.
- **무드:** 말풍선 글래스 최강(0.75). 일러스트 도트/픽셀 아트(8~16bit). 대사 16px+, 줄 높이 1.6, 선택지 터치 영역 넉넉히.

## 9. 세이브 완료
- **목적:** 오늘의 세이브 완료 축하 + 다음 액션 제시.
- **구성:** 축하 메시지(상태별 변수화) · 세이브 크리스탈 펄스 애니메이션 · 오늘 통계 카드(완료 퀘스트, 획득 칭호, 누적 회복값) · (선택) 내일 예고 · 하단 CTA "홈으로"/"기록 보기".
- **분기:** 첫 세이브 / 연속 세이브 n일 배지 / HP·칭호별 축하 문구 변화 / **0완료 세이브** 전용 위로 카피.
- **무드:** 축하 메시지 최밝은 네온, 배경 진한 퍼플. 진입 시 크리스탈 Bounce+Glow 3회, 별 파티클 흩어짐(1.5초).

## 10. 생존 기록 (도감)
- **목적:** 주간/월간 리포트 + 칭호·상태이상·회복아이템 도감 + NPC 대화 모음. 성취감 극대화.
- **구성:** 헤더(기간 선택) · 탭(주간 리포트/칭호 도감/상태이상 도감/회복아이템 도감/NPC 대화 모음).
  - *주간 리포트:* 완료 퀘스트·획득 칭호·만난 NPC·누적 회복값·연속 세이브 통계, 요일별 HP/MP 그래프.
  - *칭호 도감:* 2열 그리드, 해금/미해금("???") 카드.
  - *상태이상 도감:* 배지+경험 횟수.
  - *회복아이템 도감:* 아이콘+사용 횟수+효과.
  - *NPC 대화 모음:* NPC별 대화 기록.
- **분기:** 해금 vs 미해금(명도/투명도) · 기간 선택 시 집계 재계산 · 첫 기록 시 빈 상태 안내.
- **무드:** 카드 글래스(0.6), 도트 아이콘. 해금 시 골드 반짝임(300ms).

### 디자인 시스템 요약 (Cozy RPG)

**색 팔레트** — 따뜻한 밤(너무 까맣지 않게) + 파스텔
- 배경: 주 `#1a1620`(따뜻한 자주 어두움), 부 `#0f0d12`, 카드 분리 `#2a2230`. 그라데이션 `linear-gradient(135deg,#1a1620,#201a28)`.
- 파스텔 프로스트 패널: 기본 `rgba(240,220,240,0.12)`(라벤더), 호버 `rgba(255,220,230,0.15)`(핑크), 보조 `rgba(220,240,255,0.10)`(블루). 테두리 `rgba(255,255,255,0.08)`.
- 텍스트: 주 `#f5f0f8`(따뜻한 흰색), 보조 `#d4c5d9`, 비활성 `#8b7d95`.
- RPG 상태 색(파스텔이되 단계 구분): HP `#ff9bb5→#ff6b8a`(분홍), MP `#a8d5ff→#6db3ff`(블루), 경험치 `#ffd699→#ffb84d`(골드), 상태이상 `#f0b5a8`(살구)/`#b5d5f0`(블루), 강화 `#c4f0b5`(연두).
- 액센트: 주 `#ffb6d9`(핑크), 부 `#a8e6ff`(하늘).

**타이포** — 둥글고 친근한 결 (뻔한 기본 폰트 회피)
- 디스플레이(제목·NPC명): Poppins/Quicksand, 한글 Gmarket Sans/Gugi. 제목 32 / 소제목 24 / NPC명 18px, Bold, 줄높이 1.2, 자간 +0.5.
- 본문(대사·설명): 16px(모바일 14), 줄높이 1.6.
- UI 레이블(수치): 12~14px, Semibold.

**파스텔 프로스트 글래스**
- blur: 강 20px(모달) / 중 12px(카드·상태창) / 약 8px(호버). 알파 8~20%(표준 12~15%).
- 둥근 모서리: 6(버튼)·12(카드)·16(모달)·20px+(보스카드 등 강조), 아이콘/프로필 50%.
- 테두리 `1px rgba(255,255,255,.08)` + 이너 하이라이트 `inset 0 1px 0 rgba(255,255,255,.12)`. 따뜻한 자주 배경을 통과해 미묘한 핑크·골드 캐스트.
- ⚠️ `backdrop-filter`는 모바일에서 비싸니 영역 한정(perf-engineer 협의 필요).

**모양 언어** — 말랑·둥근
- 말랑 그림자: 기본 `0 4px 12px rgba(0,0,0,.25)`, 호버 `0 8px 24px rgba(0,0,0,.35)`, inset 움푹 `inset 0 2px 8px rgba(0,0,0,.4)`.
- 아이콘: **도트 픽셀 금지.** 2~3px 둥근 획, 통통한 프로포션(하트·별·물방울·동전 통일). 미니멀-둥근 셋.

**NPC·일러스트 방향** (치비)
- 비율 머리:몸 = 1.2:1, 눈은 얼굴 1/3~1/2(크고 반짝), 팔다리 짧고 둥글게. 선 2~3px 부드럽게, 파스텔 저채도(과자 톤), 친근한 표정.
- *일러스트 미정 시 시작 안:* 원형 실루엣+파스텔 색칠, 또는 라인 아이콘+이름 라벨(예: 여관주인 → 등불 아이콘 + "여관 주인").

**RPG UI의 귀여운 번역**
- HP=❤️ 하트+분홍 바(25% 이하 깜빡), MP=✨별/💧물방울+블루 바, 상태이상=아이콘+파스텔 배지(3개 초과 "+n"), 위험도=별점(⭐3/5) 또는 초록→노랑→빨강 바(보스는 👑+보라 배지), 칭호=둥근 파스텔 배지.

**모바일 우선:** 터치 타겟 44~48px(권장 48×56), 간격 8px+, 본문 14px+(12px 이하 금지), 제목 18px+, 간격 리듬 4/8/12/16/24px.

**애니메이션** — 말랑한 결
- 이징 바운스/스쿼시 `cubic-bezier(0.34,1.56,0.64,1)`. 지속 200~400ms. 버튼 탭=살짝 스케일다운, 카드 등장=아래→위 바운스 페이드, HP 감소=하트 떨림. `prefers-reduced-motion` 존중(모션 제거 폴백).

**글래스 위 텍스트 대비:** 주 텍스트 `#f5f0f8` on 프로스트(12%)는 WCAG AA(≥5.2:1) 충족. 바쁜 배경이면 텍스트 뒤 `rgba(0,0,0,.2)` 살짝(글래스 해치지 않으면서 대비 보강).

# 4. 팀장 종합: 게이트 점검 · MVP 로드맵 · 미결 결정

## 게이트 점검 (team-lead)
- ✅ **왜 매일 쓰는가**가 명확해졌다 — "망한 날에 켜고 싶은 앱" + 세이브=완료 아님 철학이 모든 시스템(streak·보상·DB 제약)에 박혀 있다.
- ✅ **AI 없이도 도는 뼈대**(규칙 엔진)가 있어 오프라인·AI 실패에도 앱이 멈추지 않는다.
- ✅ **가독성·대비:** 글래스 위 텍스트는 단계별 색+충분한 명도로 설계됨. 단 `backdrop-filter` 다중 패널은 모바일 성능 리스크 → 구현 전 perf-engineer 점검 필수.
- ✅ **접근성:** `prefers-reduced-motion` 명시. 입력 폼·터치 타겟 기준 확보.
- ⚠️ **일관성 지적:** designer 시안의 스탯 라벨("매우 좋음/괜찮음/피곤함/힘들어")과 규칙 엔진의 라벨("거의 쓰러짐/버티는 중/그럭저럭/꽤 괜찮음")이 다르다. **규칙 엔진 라벨로 통일**(서브컬처 톤이 더 살고 점수 매핑과 1:1).

## MVP 로드맵 (가장 작은 합리적 범위부터)

**P0 — 핵심 루프 (먼저 완성):** 스플래시 · 온보딩 · 체크인 · 상태창 결과 · 퀘스트 · 세이브 완료 + 홈. 규칙 엔진 + 폴백 문구(템플릿)만으로 **AI 없이** 전체 루프가 돌게 만든다. Supabase 인증·저장.

**P1 — AI 레이어:** LLM 연결(퀘스트 문구=가벼운 모델, NPC 대사=풍부한 모델) + NPC 상담소·대화 화면. 구조화 출력·캐싱·실패 폴백·안전 모더레이션(§5-2).

**P2 — 리텐션:** 생존 기록·도감·별사탕·streak 마일스톤·테마/스킨.

# 5. 리뷰 보강 (Hardening) — 2026-06-10 팀 비판 리뷰 반영

팀 3인(developer·ideator·designer)이 기획서를 비판적으로 통독해 발견한 구멍. **🔴 구현 착수 전 반드시 못 박을 것** / **🟡 MVP 이후 보강.**

## 5-1. 데이터·로직 (developer)
- 🔴 **"오늘" 날짜 경계 단일 정의** — *오늘 = 프로필에 저장된 IANA 타임존 기준 local date.* 모든 날짜 컬럼·streak·UNIQUE 제약·서버 함수가 이 기준 하나만 사용. (자정 넘김·여행 시 streak 오작동·이중 체크인 방지)
- 🔴 **RLS + 인덱스** — 모든 유저 테이블 `auth.uid() = user_id` RLS(읽기/쓰기 분리), `collectibles` 공개 읽기. `(user_id, *_date desc)` 복합 인덱스. (RLS 없으면 anon 키로 전체 데이터 노출)
- 🔴 **게스트→계정 병합 = 행 union 후 집계 재계산** — 체크인/세이브 행을 날짜 union(서버 우선)으로 합친 뒤 streak·별사탕·도감을 **원천 데이터로 재계산**. `user_collectibles`는 `ON CONFLICT DO NOTHING`. (단순 합산 시 streak 부풀림·PK 충돌)
- 🔴 **오프라인 정합성 = 서버 재계산이 단일 진실** — 세이브는 멱등 키(`user_id+save_date`)로 큐잉, streak/별사탕은 서버 재계산값을 확정값으로 표시. 오프라인 중엔 "동기화 후 확정" 잠정 표기. UPSERT 멱등.
- 🟡 difficulty 반열린구간화(반영 완료, §2-2), 입력 1–4↔점수 0–3 변환 규칙 고정.
- 🟡 금지 퀘스트 `violated bool` 컬럼(반영 완료, §2-4).
- 🟡 다중 보스·미완료 이월은 **MVP 스코프 아웃**(단일 보스). 추후 `bosses` 테이블 분리 여지만.

## 5-2. LLM 안전·비용 (developer)
- 🔴 **입력 검증 + 출력 모더레이션** — `boss_text`·`nickname`·`job_alias` 길이/문자 제한, 사용자 입력은 시스템 프롬프트와 구획 분리(인젝션 방어). 출력은 자해/혐오 분류 통과 후 표시, 위반 시 **안전 폴백 대사 강제.** (정신건강 맥락 — 자해 신호에 라이벌이 도발하면 안 됨)
- 🔴 **호출 상한 + 캐시** — 유저별 일일 LLM 호출 캡(덮어쓰기·리롤 포함) + 캐시 재사용으로 비용 폭주 차단.
- 🟡 **`unlock_rule` jsonb 스키마 명세** — 예: `{type:'streak'|'tag_count'|'mode_first', op:'>=', value:N}`. 평가는 **서버 단일 주체**(트리거/RPC, 멱등 INSERT), 동기화 시 전체 룰 재평가 1회.

## 5-3. 리텐션·제품 (ideator) — "닫혀 있는 23시간"을 설계
- 🔴 **알림 = NPC가 말 거는 방식** — 온보딩 마지막에 "NPC 알림 시간" 옵트인 설정(예: 힐러가 밤 9시 "오늘 체크인 했어?"). Capacitor 푸시, 세계관 톤 유지. 끌 수 있게. (d1→d7 리텐션 최대 변수)
- 🟡 **컴백 루프** — streak 끊김을 "실패"로 안 보이게. 복귀 체크인 시 NPC 전용 대사("N일 만이네, 어떻게 지냈어?"). `saves`의 gap_days를 LLM 컨텍스트로(스키마 변경 불필요).
- 🟡 **2일차 브릿지** — 2일차 홈에 "어제의 [닉네임]이 남긴 기록" + 어제 NPC 대사 한 줄 리마인드로 어제-오늘 연결.
- 🟡 **무기력 방지 균형추** — 생존 모드 0완료 세이브 3회 연속 시 힐러가 비난 아닌 호기심 톤으로 가볍게 개입("요즘 계속 회복 중이네, 하나만 물어봐도 될까?").
- 🟡 **차별화 위계 재설정** — "아무것도 못 해도 streak이 유지되는 유일한 루틴 앱"을 단일 핵심으로 전면, 금지 퀘스트는 2순위. (포폴 데모 첫 30초 메시지)
- 🟡 **세계관 밀도 한 끗** — NPC 소개 카드 한 줄 배경 서사, 세이브 크리스탈 내러티브("오늘을 담는다, 깨져도 기억은 남는다"), 상태이상 세계관 용어 오버레이(선택 표시).
- 🟡 **포트폴리오 의도 명시** — 외부 리뷰어용 메타 서술 2~3문장(기술 의사결정 3가지: 규칙엔진·LLM 경계·zero-failure streak + 제품 철학).

## 5-4. 디자인·접근성 (designer)
- 🔴 **색각이상 대응** — 정보를 색만으로 구분 금지. 상태이상·HP·위험도·금지에 **아이콘+텍스트 라벨 필수.** 금지 퀘스트는 빨강 대신 **앰버/황색(주의 톤)**으로 위험도 적색과 분리. 위험도는 ⚠️ 아이콘·테두리·해치 패턴 보강.
- 🔴 **네비게이션 맵** — P0 선형 루프: 홈→체크인→상태창→(퀘스트∥NPC)→세이브. 뒤로가기: 체크인/상태창/퀘스트는 이전 화면(미저장 입력 폐기 경고), 대화는 NPC 상담소로, 세이브 후는 홈 강제 이동. 기록은 하단 탭(언제든 진입/이탈).
- 🔴 **빈/로딩/에러 상태** — 화면별 3종 정의. AI 생성 중=골격+타이핑 `...`, 폴백=자연스러운 기본 대사, 빈 기록=일러스트+"오늘 첫 세이브를 하면 여기 나타나요". (3초 타임아웃 UI 명시)
- 🟡 **무드 노트 도트→둥근 정렬** — 스플래시 "도트 입자"→말랑 원형 입자, 상태창 "도트 테두리"→둥근 12px+프로스트 테두리, 세이브 완료 "별 파티클"→파스텔 바운스. (§3 상단 치환 원칙 + 와이어프레임 시 확정)
- 🟡 **MVP NPC 범위 UI 반영** — 온보딩 선호 선택은 사용 가능 2종만, 나머지 3종은 "[예정]" 배지/도감 잠금. NPC 상담소 "모두" 필터도 사용 가능 NPC만.
- ✅ **테마 정책 = 다크 전용(MVP)** — '밤의 여관' 컨셉 유지. 라이트모드는 v1.2 로드맵(파스텔 유지·배경 명도만 역전).

> **반영 상태:** 🔴 항목은 P0 착수 전 스키마/플로우에 반드시 반영. 🟡 항목은 해당 단계(P1/P2) 진입 시 처리. difficulty 반열린구간·`violated` 컬럼·RLS 노트·NPC 폴백은 본 문서에 **이미 인라인 반영 완료.**
