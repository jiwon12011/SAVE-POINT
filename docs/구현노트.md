# SAVE POINT — 구현 노트 (P0 프로토타입)

> 기획 문서가 실제로 돌아가는 프로토타입이 됐을 때의 구현 기록. 코드 구조·실행법·UI 구조 결정·검증 방법.

## 1. 개요
- **형태:** 바닐라 HTML/CSS/JS 단일 페이지 프로토타입 (사용자 실제 쇼케이스 스택과 동일).
- **상태:** P0 핵심 루프가 **데이터로 실제 동작** — 체크인 → 규칙 엔진 → 상태창·퀘스트·NPC 대사·세이브.
- **실행:** `index.html` **더블클릭만 하면 됨**(서버 불필요). 데이터를 `data.js`로 임베드해 `fetch` 의존을 없앰.

## 2. 파일 구조
| 파일 | 역할 |
|---|---|
| `index.html` | 화면 구조 (홈 + 모달 팝업 + 하단 네비) |
| `styles.css` | Cozy RPG 디자인 시스템 (파스텔 프로스트 글래스, 다크 테마) |
| `engine.js` | **규칙 엔진**(순수 함수) — 상태→energy→모드, 추천 NPC·표정·대사 카테고리, 퀘스트 빌더. node에서 단위 검증 가능 |
| `app.js` | DOM 로직 — 홈 NPC 상호작용, 모달 열기/닫기, 렌더링, 세이브 |
| `data.js` | 폴백 데이터(퀘스트·대사) 임베드 — `docs/data/*.json`에서 생성 |
| `assets/` | NPC 스프라이트(표정별)·상태이상·배지·배경·아이콘 |
| `docs/data/` | 데이터 단일 진실원(JSON) — `data.js`의 출처 |

## 3. UI 구조 결정 (핵심)
> 사용자 피드백으로 **두 번 갈아엎은** 부분. 기록해 둘 가치가 있음.

- **1차:** 5개 화면을 동등하게 탭 전환 → *"팝업만 계속 뜨는 느낌, 흐름이 없다"*.
- **2차:** 체크인 저장 → NPC 대화 장면(비주얼노벨식) → 퀘스트, 선형 흐름 → 게임 느낌은 살았으나 홈이 여전히 상태창으로 꽉 참.
- **최종(현재):** **홈 = NPC가 배경에 서 있는 "내 공간"** + **체크인·상태창·퀘스트·세이브 = 버튼으로 뜨는 팝업(모달).**
  - 홈 NPC를 **탭하면 대화**(타자기 효과). 대사를 넘길 때마다 **표정도 바뀜**(진단=상태표정 → 위로=comfort/cheer → 마무리=relax/joy).
  - 핵심 깨달음: *팝업이 어색했던 건 팝업 자체가 아니라 **돌아갈 베이스(홈)**가 없어서였다.* 홈이라는 거처가 생기니 버튼→팝업이 자연스러워짐.

## 4. 게임풍 연출 & 시스템
- **RPG 대화창:** 이름판 + 한 글자씩 타자기 + (대화창 탭 시 즉시 완성).
- **표정 매칭:** `engine.computeStatus`가 모드/HP로 표정 슬롯(`default`/`comfort`/`joy`/`cheer`/`relax`)을 결정 → `app.npcSprite`가 스프라이트로 변환. 슬롯 없으면 폴백(예: `innkeeper-cheer` 부재 → `joy`). 홈 NPC는 대사를 넘길 때마다 표정도 전환.
- **성장(레벨/EXP):** HUD에 Lv + EXP 게이지. 퀘스트 완료 +5 EXP, 세이브 +20(+완료당 10). 게이지가 차면 레벨업 연출(`lvpop`).
- **재화(별사탕):** HUD 별사탕 카운터. 퀘스트 완료 시 `+5 ⭐` **플로팅 보상 연출**, 세이브 시 적립.
- **수집(도감):** 생존 기록 모달 — 연속 세이브·통계 + **칭호 배지 12종 도감**(획득/잠금 그레이스케일). 세이브 조건에 따라 배지 해금(`unlockBadges`).
- **모달 등장:** 살짝 떠오르는 pop 애니메이션 + 딤 배경.
- `prefers-reduced-motion` 시 타자기·모션 생략.

## 5. 데이터 흐름
```
체크인 입력 (HP/MP/집중 1~4 + 상태이상 + 보스)
  → engine.computeStatus → { energy(0~9), mode, title, recommendedNpc, expression, category }
  → engine.buildQuests(모드, 보스) → 메인/서브/금지 (토큰 {boss}/{time_box} 치환)
  → engine.pickDialogue(npc, category) → 폴백 대사 1줄
  → 홈 NPC 갱신(표정·대사) + 상태창/퀘스트 모달 렌더
```

## 6. 검증 방법 ("수정→점검" 루프)
- **엔진:** `node`로 시나리오 테스트(생존/보통/도전/결정장애 등 → 모드·퀘스트·대사 확인).
- **DOM:** `jsdom`으로 실제 클릭 흐름 구동(홈 NPC 탭→대화/표정, 체크인 모달→저장→홈 갱신, 상태창·퀘스트·세이브 모달, 보스 치환).
- **에셋:** CSS/HTML/스프라이트맵이 참조하는 파일이 모두 실재하는지 교차 검사.

## 7. 알려진 한계 / 다음
