# SAVE POINT — 포션 연금술 시스템 기획서 v2.0

> 지친 하루를 RPG 상태창으로 번역하던 SAVE POINT 위에, **마법사의 연금술 공방**을 얹는다.
> 오늘의 컨디션이 재료가 되고, 작은 실천이 포션으로 결정된다. **망한 날도 레시피가 된다.**
> v2.0: 재료를 **5축 57종**으로 확장 + **씨앗→재배→수확** 경제 추가(씨앗 수급 NPC·퀘스트·출석·구매 4채널).

작성: jiwon-harness 팀 (team-lead · ideator · developer) · 2026-06-10
진실원: `docs/data/ailments.json` · `docs/data/collectibles-rewards.json` · `assets/`
이 문서는 **기획·설계 명세**다. 구현은 이 문서를 기준으로 들어간다.

## 0. 확정 사항 (이 위에서 모든 설계가 이뤄진다)

| 항목 | 결정 | 비고 |
|---|---|---|
| 피벗 방향 | **A안 — 자기돌봄 Why 계승 + 포션 메타포 레이어** | 순수 게임(B안) 아님. 기존 자산 재활용 |
| **불가침 원칙** | **포션 제작은 세이브의 *보상*이지 *조건*이 아니다** | 재료 없어도/포션 안 만들어도 세이브는 된다. "실패=손해" 구조 금지 |
| 콘셉트 한 줄 | "오늘의 컨디션이 재료가 되고, 작은 실천이 포션으로 결정되는 — 망한 날도 레시피가 되는 연금술 일지" | |
| 무드 | 기분 좋은 어수선함. 마법사 작업실의 온기 · 파스텔 · 따뜻한 밤 · 글래스 프로스트 | 사용자 취향(화려·코지) 존중 |
| 재화 | **별사탕(stardust) 단일** | 새 화폐 만들지 않음 |
| NPC | **5명 전원 의뢰 발주자** (사용자 지시) | healer·innkeeper·guildmaster·rival·wizard. 마법사=연금술 멘토 승격 |
| 재료 | **5축 57종 + 씨앗 재배** (사용자 지시: "아주 다양") | A 컨디션결정체 11 · B 실천결정체 7 · C NPC흔적 5 · **D 재배작물 26** · **E 자연채집 8**. §14~17 |
| **씨앗·재배** | **씨앗→텃밭에 심기→세이브할 때마다 한 단계 성장→수확** | **실시간 타이머 금지**(앱 안 열어도 안 시듦). 씨앗 수급: NPC·퀘스트·출석(streak)·구매 4채널 |
| **재배는 보상이지 조건 아님** | **재배 안 해도 기본 포션 11종 전부 완성** | 재배 작물(cropBonus)은 "더 쉽게 완벽"이지 "완벽의 조건"이 아니다. 대체조합으로도 완벽 도달 가능 |
| **텃밭 = 1차 풀스크린** (사용자 결정) | **텃밭을 1차 풀스크린으로 승격** — 재배 경험 풀볼륨 | 사용자가 화분 위젯 대신 풀스크린 텃밭 1차를 명시 선택. 신규 풀스크린 **3개**(공방·도감·텃밭). 상점(씨앗 구매)은 2차 유지 |
| 신규 시스템 한계 | **신규 화폐 0·신규 데이터 마스터 0** (풀스크린은 3개로 상향) | 씨앗 id만 신규 데이터. 스코프 가드: 텃밭 1차는 **세이브 기반 성장·플롯 3~4칸·상점 제외**로 한정 |
| MVP 1차 | **공방(휘젓기) + 포션 도감 + 텃밭(재배) + 초경량 의뢰 1건 + 씨앗 4채널 수급** | 상점·5인 의뢰·플롯 확장(9칸)은 2·3차 |

### 핵심 게임 루프
```
[체크인·세이브] ──재료 드롭 + 씨앗 드롭 + 텃밭 한 단계 성장──┐
                              ▼
[텃밭(1차): 씨앗 심기·재배·수확] ─┐    [공방: 휘젓기 제작]
[상점:씨앗 구매(2차)]          ─┴─재료 인벤토리─→  │
                              ▼
              [포션 도감 채우기] + [NPC 의뢰 납품]
                              │
                        별사탕 획득 ──→ 재료/씨앗 (순환)
```

## 1. 왜 — 차별화

Finch·Habitica는 "성장의 보상"으로, 방치형 포션게임은 "시간의 보상"으로 포션을 준다.
SAVE POINT는 **"오늘의 컨디션 자체가 레시피"** — 좋은 날이 아니라 **힘든 날이 더 희귀한 재료를 드롭하는 유일한 게임.** 망한 날일수록 도감이 더 빠르게 채워진다.

이것이 기존 영혼("아무것도 못 한 날에도 닫지 않는 앱")과 정확히 맞물린다. 포션은 자기돌봄을 대체하지 않고 **더 매력적으로 시각화하는 레이어**다.

## 3. 재료 카탈로그 (5축 57종 — 중복 데이터 없음)

> v2.0: 사용자 지시("재료 아주 다양 + 씨앗 재배")로 **A·B·C 3축(23종) → A~E 5축(57종)**으로 확장. D 재배작물·E 자연채집은 **§14~17**에 상세. 여기 §3은 1차 코어인 A·B·C축을 정의한다.

| 축 | 분류 | 수량 | 획득 | 1차 |
|---|---|---|---|---|
| A | 컨디션 결정체 | 11 | 체크인 태그 자동 드롭 | ✅ |
| B | 작은 실천 결정체 | 7 | 퀘스트 완료 | ✅ |
| C | NPC 흔적 | 5 | NPC 대화 드롭 | ✅(2종) |
| D | 재배 작물 | 26 | 씨앗 심기→세이브 성장→수확 | ✅(화분 위젯) |
| E | 자연 채집물 | 8 | 시간대·조건 드롭 | ✅(void_echo 1종) |

재료 id는 **단일 네임스페이스를 공유**한다. A축=ailment id, B축=recoveryItem id, C축=NPC 흔적 id, D축=cropId, E축=채집물 id. 신규 데이터 마스터는 만들지 않고(unlock_rule·인벤토리 구조 재사용), 새로 생기는 id는 **씨앗(`seed_*`)·작물(`cropId`)·채집물뿐**이다.

| `focus_lost` | 집중의 결정체 | 집중력 실종 태그 선택 |
| `decision_paralysis` | 미궁의 결정체 | 결정 장애 태그 선택 |

`recoveryItems`의 id를 재료로 승격. **별도 정의 없음.** 퀘스트 완료 시 드롭(아래 §6 드롭 규칙).

| id (= materialId) | 한글명 | 해금 조건(기존 unlock_rule) |

### C축 — NPC 흔적 (5종) · "연결이 재료가 된다"
NPC 대화 시 드롭. **1차는 "NPC 전환 시 1회 보장 드롭"**(결정론 원칙 유지). 확률 드롭은 P2.

|---|---|---|---|

| `trace_wizard` | 마법사의 별가루 | wizard | N (P2) |

### 압박 방지 장치
**모든 레시피에 필수 재료가 없다**(와일드카드 제외). 추천조합 → 고품질, 보유 재료만(대체조합) → 무조건 완성, 품질만 변동. 재료 부족은 "못 만든다"가 아니라 "오늘 가진 것으로 어떤 버전이 나오나"가 된다. 도감을 채우려고 일부러 나쁜 컨디션을 고를 이유도 없다(이미 겪은 컨디션이 재료라서).

## 5. 공방 — 휘젓기 제작 & 품질

### 제작 흐름 (3단계)
1. **재료 넣기** — 보유 재료 칩을 플라스크에 탭/드래그. 레시피 일치 시 파스텔 글로우. 불일치여도 진행됨(품질만 변동). "마법사 멘토 힌트" 버튼(무료, 하루 3회).
2. **휘젓기** — 손가락으로 원을 그리며 젓기. 시계/반시계 번갈아. **일정한 리듬**이 좋은 결과(빨리 많이 ≠ 좋음). 리듬 = "지금 이 순간 집중하기"의 작은 메타포.
3. **완성** — 플라스크가 포션 병으로. 파스텔 파티클 + 도감 선반에 꽂히는 연출. **완성 = 오늘의 세이브 트리거.**

### 품질 3등급 (무압박 — "실패" 없음)
| 등급 | 라벨 | 아이콘 | score |
|---|---|---|---|
| 평범 | 기본 포션 | 회색 물방울(작게) | 0~3 |
| 좋음 | 숙성 포션 | 파란 물방울(테두리) | 4~6 |
| 완벽 | 명품 포션 | 황금 물방울 + 빛 파티클 | 7~10 |

> 색만으로 구분 금지: **라벨 텍스트 + 아이콘 형태(크기·테두리)** 병기.

### 점수 산출식
```
품질 점수 = (추천재료 일치 수 × 2) + (휘젓기 리듬 안정도 × 4)
  · 추천재료 일치 수 0~3 → 최대 6점
  · 리듬 안정도 0.0~1.0 → 최대 4점 (자동제작/REDUCED 시 0.5 고정 = 2점)
  최대 10점.  평범 0~3 / 좋음 4~6 / 완벽 7~10
```

### 접근성 — REDUCED 경로를 *먼저* 설계
`prefers-reduced-motion` 또는 드래그 불가 사용자: **"제조하기" 버튼 1탭 = 즉시 제작.** 품질은 재료 충족도로 결정(A·B 둘 다=완벽, 한 축=좋음, 최소=평범). 모션 경로는 이 위에 얹는다 — 모션이 빠져도 제작은 100% 동등.

## 6. 드롭 훅 (engine.js 무변경)

단일 헬퍼 `addMaterial(id, n)`로 통일, 기존 3함수에 각 1줄.

| 축 | 트리거 | 삽입 위치 | 동작 |
|---|---|---|---|
| A | 체크인 저장 | `applyCheckin()` 끝 | 선택한 ailment마다 `addMaterial(id,1)` |
| B | 퀘스트 완료 | `addQuest` 보상 블록(`rewardedKeys` 가드 안) | **그날 첫 ailment의 `recommendedRecovery` 지급. ailment 없으면 `water`.** (확정) |
| C | NPC 대화 | NPC 전환 시 | **1회 보장 드롭**(확률 X, 1차). healer·innkeeper만 활성 |

`persist()`는 보상 변동 분기 끝에서 1회만(addMaterial은 persist 안 함).

## 9. 데이터 모델 & 구현 경계 (개발)

기존 패턴(평면 state · 선택적 persist · `openModal`→`render*` · createElement 루프 · 상단 1회 REDUCED) 계승. **engine.js 무변경.**

### 신규 state 슬롯
```js
materials:     {},   // { materialId: count } — 3축 단일 네임스페이스
potionCodex:   {},   // { potionId: { firstAt, bestQuality:0|1|2, count } }
activeRequest: null  // { id, npc, wants, reward, status:'open'|'done' }
```

### 영속 마이그레이션 (구버전 안 깨짐)
```js
// restore() 복원부
state.materials     = s.materials     || {};
state.potionCodex   = s.potionCodex   || {};
state.activeRequest = (typeof s.activeRequest === "undefined") ? null : s.activeRequest;
```
`SAVE_KEY` 유지(`savepoint_state_v1`). 머지 방어로 버전 안 올림.

### data.js 확장
`window.SAVEPOINT_DATA`에 `potions` / `recipes` / `recovery`(recoveryItems 임베드) 추가. data.js는 **생성 파일** → 소스 JSON(`docs/data/potions.json` 신규)에서 재생성. 부팅 시 `buildMaterialInfo()`로 `Engine.AILMENTS` + recoveryItems를 합쳐 표시용 룩업 맵 생성(중복 정의 없이 참조만).

### 신규 렌더 함수
| 함수 | 따라갈 패턴 | 역할 |
|---|---|---|
| `renderWorkshop()` | `renderQuests()` | 재료 칩 + 제작 가능 포션 + 휘젓기 영역 |
| `renderPotionCodex()` | `renderRecords()` badge-grid | 12종 그리드, 해금=키 존재 |
| `renderRequest()` | `renderQuests()` boss 카드 | 의뢰 1건 카드, 충족 시 납품 버튼 |

### 휘젓기 성능 (perf-engineer 게이트)
- 입력: `pointermove`에서 **각도만 push**, 계산·렌더는 **rAF 단일 루프**(레이아웃 스래싱 회피).
- 비싼 지점: 파티클 **개수 캡 + 풀링**, 트레일 `backdrop-filter`/blur 영역 한정, `will-change` 신중.
- **동작본 나오는 즉시 perf-engineer가 모바일 60fps 검증** — 통과 못 하면 연출 강도 조정.

## 15. 씨앗 시스템 (세이브 기반 성장 — 압박 없음)

- 씨앗 id = `seed_{cropId}` 1:1. 씨앗 보유 시 텃밭 플롯(plot)에 심기 → 씨앗 1개 소모.
- **성장 트리거 = 세이브(체크인) 1회당 1단계 (실시간 타이머 금지).** 앱 안 열면 그대로 멈춤, 시들지 않음.
- 빠름 3세이브 / 보통 5 / 느림 8 만에 수확 가능. "n번 더 기록하면 열매를 맺어요" 표시.
- **텃밭 플롯(plot)**: **1차 풀스크린**(사용자 결정). 1차 시작 **3~4칸**. 확장 — 첫 수확 +1, save_count 30 +1, 100 +2, 전설 첫 수확 +2(최대 9칸)는 2차. 1차는 씨앗 구매(상점) 제외 — NPC/퀘스트/출석으로만 수급.

```js
// state.plots: 최대 9개
{ plotId, cropId|null, growthProgress, growthRequired, plantedAt, harvestReady }
// 세이브 시 growthProgress++; 도달 시 harvestReady=true. 수확 탭 → material+1, plot 초기화
```

## 17. 희귀도 ↔ 품질 연결 + cropBonus

| 등급 | 입수 | 포션 영향 |
|---|---|---|
| 흔함 | 항상 | 기본~좋음 |
| 고급 | 퀘스트·NPC | 좋음 안정 |
| 희귀 | 마일스톤·상점 | 완벽 주재료 |
| 전설 | 장기 streak·100일 | 히든 전용 + 완벽 보장 |

**cropBonus** — 레시피(§4)에 재배 작물 추가 시 점수 가산(예: `dream_fog_elixir` ← moonherb+1/dreambell+2/nightbloom+3). **원칙: 재배 없이 대체조합으로도 완벽 도달 가능. cropBonus는 "더 쉽게 완벽"이지 "완벽의 조건"이 아니다.** 예외 1건: 히든 `black_night_tincture`만 전설작물(dream_crystal_flora) 필수 — 의도된 도전.

## 19. 압박 방지 3장치 (재배)

1. **시들지 않는 화분** — 수확 준비 후에도 사라지지 않음. 독촉 문구·빨간 뱃지 금지. 알림 기본 off.
2. **재배 없이도 포션 완성** — A+B 조합만으로 기본 11종 전부 제작. 화분 탭 안 열어도 도감 완성 가능.
3. **씨앗은 쌓인다** — 유효기간 없음. 카운트다운 UI 금지. 씨앗 서랍은 그냥 서랍.

## 20. 작물 에셋 전략 (designer)

작물은 신규 콘텐츠(기존 에셋 0장). 기존 113장은 **384×384 PNG, 파스텔 코지 톤**. **풀세트 78장 대신 티어드 ~36장**으로.

### 총량 — 티어드
| 구성 | 수량 | 용도 |
|---|---|---|
| 작물별 수확형 아이콘 | 26 (1차 12 / 2차 14) | 인벤토리·도감 카드·수확 연출 |
| 성장 공용 실루엣 | 분류 5종 × 3단계(새싹/자람/수확직전) ≈ 15 | 텃밭 성장 표시. 희귀도 틴트 재사용 |
| 씨앗 봉투 | 분류별 6 + 라벨 | 인벤토리(개별 26장 대신) |
| **합** | **~36장** (1차 ~22 / 2차 ~14) | 풀세트 대비 제작 54%·번들 45% 절감 |

### 규격·일관성
- **384²** 유지(도감·수확), 인벤 칩은 **96²** 다운스케일. **투명 배경**(플롯에 떠 있는 느낌).
- 팔레트: `ailments.json` 색감(보라·하늘·초록·따뜻한 갈색·은색). 주조색 1~2 + 하이라이트 1.
- 기존 ailment 일러와 **같은 선 굵기·드롭섀도우·hand-drawn 톤**. 1~2px 다크브라운 외곽선.
- 성장 공용 실루엣은 분류별 동일 라인/섀도우 규칙, **틴트만 변동**.

### 1차 필수 작물 12종 (healer·innkeeper·초기 streak·퀘스트)
`moonherb` · `dreambell` · `clarity_moss` · `echo_thyme` · `still_lavender` · `dew_berry` · `marsh_root` · `foxfire_cap` (필수 8)
\+ 옵션 `compass_vine` · `signal_grass` · `ghost_pepper_mild` · `void_lily` (4). 고급·희귀·전설 14종은 2차.

### 제작 — AI 일관성 (기존 톤 드리프트 방지)
- 공통 프롬프트: `cozy fantasy illustration, soft pastels, warm night ambiance, matte/hand-drawn, 1-2px dark-brown outline, soft drop shadow, transparent background, 384x384, single centered item`
- 작물별 추가 예: `moonherb: silvery nocturnal herb, dreamlike glow` / `dreambell: pale bell-shaped flower, soft chime aura` / `foxfire_cap: small glowing mushroom, ember light` / `marsh_root: muddy twisted root, humble`
- 배치: NPC 담당별 묶음 생성 → 기존 `ailment-anxiety.png`와 나란히 톤 QA → hue/sat 후보정. 희귀도 변형은 Curves 매크로로 자동화.

> 결론: **새로 필요하지만 1차는 ~22장.** 풀세트 안 그려도 "아주 다양"은 인벤·도감의 개별 아이콘으로 충족되고, 텃밭 성장은 공용 실루엣으로 경제적.
